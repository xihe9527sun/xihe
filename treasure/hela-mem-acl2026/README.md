## 第一遍：去粗取精 | 核心精华

### 核心贡献
HeLa-Mem将对话历史建模为带**赫布学习动力学**的动态图。三个生物学机制：关联(赫布学习)、巩固(反思蒸馏)、扩散激活(联想检索)。

### 赫布学习公式
$$w_{ij}^{(t+1)} = (1-\lambda) \cdot w_{ij}^{(t)} + \eta \cdot \mathbb{I}(v_i, v_j \in \mathcal{K}_t)$$
- η=0.02（共激活增量），λ=0.995（衰减率），τ=60天（时间常数）

### 曦和最该抄的三个设计
1. **扩散激活检索** — 不是只查Top-K语义相似，还沿着赫布强边"联想"过去。语义远但关联近的记忆不会被漏掉
2. **反思性蒸馏阈值** — `D(vi)=Σwij > δhub` → 触发LLM合成语义知识。赫布权重不只用于检索，还驱动"知识结晶"
3. **双路径检索** — 语义路径+赫布扩散路径并行，F1提升5.71（时序任务）

## 第二遍：去伪存真 | 检验与质疑

**质疑：** 赫布学习在记忆系统里真的有用吗？还是花哨的生物学类比？

**检验：** 消融实验给出了硬数据——去掉扩散激活F1降-2.55，去掉反思Agent降-4.87。赫布路径不是装饰，是实打实的信息检索增强。而且它的Token效率极高（~1010 vs 基线2000+）。

**结论：** 赫布学习在Agent记忆场景下站得住。不是生物学噱头。

## 第四遍：由表及里 | 架构层洞见

**表层：** 一篇ACL 2026的Agent记忆论文，讨论赫布学习的工程实现
**底层：** 曦和的超图系统有架构基础但缺乏动力学——边权重不会自己长大也不会自己消失

**核心定理：**
> 赫布权重不应该是静态记录，而应该是动力系统。

曦和现在的赫布追踪器在记录"哪条路径走过多少次"，但权重不会随时间衰减，也不会在共激活时自动增长。没有动力学的赫布图只是一张静态快照。

### 今天就该反哺的一条

把代谢路由的**赫布权重更新逻辑**从"路径被选中→加1"升级为HeLa-Mem公式：
```python
# 旧逻辑（静态计数器）
path_hits[path_id] += 1

# 新逻辑（赫布动力学）
for pair in coactivated_pairs:
  w[pair] = (1-0.005) * w[pair] + 0.02  # 衰减+增长
```
