## 一、核心发现：反事实推理的三步流程

论文对Pearl因果阶梯(Ladder of Causation)第三级——反事实推理——做了完整的工程化实现：

```
反事实推理 = Abduction(隐变量推断) + Intervention(替代情境构建) + Prediction(结果预测)
```

关键发现：现有评估方法**跳过Abduction步骤**，实际退化为干预推理(第二级)，导致高估LLM能力25-40%。

## 二、核心发现：RL训练 > SFT

**关键实验对比：**

| 训练方法 | 分布内性能 | 分布外泛化(代码→数学) |
|:---------|:---------:|:-------------------:|
| SFT(强模型轨迹) | ↑ 提升 | ↓ 下降 |
| RL(PPO) | ↑ 提升 | ↑ 提升 1.5x-2x |

这一发现是曦和这篇论文中**最有价值的养分**——曦和的代谢进化路径目前依赖H值自然选择，本质上接近SFT的"在成功路径上微调"。RL训练能够诱导核心认知行为并泛化到新分布，这对代谢路由的进化机制是一个修正信号。

## 三、技术方法

### 可执行反事实框架
- 以代码(Code)和数学(Math)问题作为操作化载体
- 修改if-condition/while-loop等结构生成不同难度反事实场景
- 代码执行提供**确定性因果推理基板**——不需LLM猜测，直接执行验证

### 可扩展合成数据生成
- 通过修改代码结构自动生成不同难度数据
- 支持从简单→困难的渐进式训练
- 零人工标注成本

### 推理轨迹分析
- SFT导致推理轨迹变得"狭窄"——模型学习的是特定的推理格式而非推理能力
- RL保持推理轨迹的多样性，从而支持泛化

## 四、对曦和的意义

### 直接融合方向

1. **代谢进化路径 → RL-like泛化奖励** (P0)
   - 现状：代谢路由的H值进化接近SFT式同路径强化
   - 改变：在surprise_credit中加入"跨域泛化奖励"——当一个路径的非原生域命中时，surprise_credit加权×1.5
   - 预期效果：代谢系统从"走同一路径更快"变为"走新路径也有回报"

2. **执行核 → 可执行反事实验证** (P0)
   - 现状：causal.crs和causal.minimal_repair的修复策略是启发式的
   - 改变：添加"代码化反事实验证"步骤——将修复建议编码为可执行代码片段并运行验证
   - 预期效果：修复决策从概率正确变为确定性正确

3. **桥接简报 → 反事实自信度** (P1)
   - 现状：桥接输出皮层状态但不提供因果确信度
   - 改变：在session-context中增加'反事实自信度'指标——当给出因果判断时，同时输出"如果改变X会怎样"
   - 预期效果：盘古能看到曦和推理的因果粒度
