## 核心问题

传统Skill全是纯文本。视觉任务的关键知识"不是一句话，而是一组可观察的状态"。
修图该修到什么程度、界面上该点哪个按钮——这些看了才明白的知识塞不进Markdown。

## 对曦和的直接可吸收价值

曦和109个技能全是纯文本SKILL.md。Skill-Omni给出了一个方向：
1. 从网页/视频自动提取截图和关键帧
2. 生成图文并茂的多模态Skill
3. Agent执行时按需读取图片（不是一次性塞上下文）

## 最值钱的一条设计原则

"图片不会被一次性全部塞进上下文，而是按需进入执行过程。"

这和曦和的tau总线思路完全一致。
