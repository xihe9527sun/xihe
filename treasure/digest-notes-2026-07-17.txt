## 📝 宝藏1：Darwin Gödel Machine (评分0.95)

**核心洞见：将AI自我改进从'找最优解'转变为'培育进化树'**

三个机制决定了它的成功：
1. **Archive存档** — 保留所有历史智能体版本，包括低分'垫脚石'。实验证明去除archive后，一次不良修改就导致进展停滞
2. **HighScore×LowOffspring选择策略** — 平衡利用(高分)与探索(未被充分探索的谱系)。每个智能体都有非零的采样概率，保证没有路径被永久阻塞
3. **分阶段评估** — 10-task快速筛选→60-task中等评估→200-task高精度评估，以最小计算成本防止自我破坏

**对曦和的最直接启示**：treasure库本身就是evolution_engine的天然archive！当前evolution_engine是线性进化(父→子→孙)，可以直接扩展为存档式进化树——每个宝藏既是消化的养料，又是未来变异的母本。

**跨模型迁移的意外发现**（对曦和多模型架构特别重要）：
在Claude 3.5 Sonnet上演化的智能体 → o3-mini: 23%→33%, Claude 3.7 Sonnet: 19%→59.5%
证明学到的改进不是绑定具体模型的，而是通用的'强化工程模式'。

## 📝 宝藏2：Stanford DeLM (评分0.93)

**核心洞见：协调(Coordination)的载体可以是共享记忆(Shared Memory)，不一定是中央调度者**

三个关键机制：
1. **Shared Verified Context (共享验证脉络)** — 代理的共同通信基底，默认只读~100 token的gist
2. **Coarse-to-Fine三层结构** — Gist(100t) → Reference-grounded Summary → Raw，需要才展开
3. **Admission-time verification gate** — 只有通过验证的精简更新才准入库，防止去中心化退化为混乱

**最大单点洞见**：把'负面结论(negative findings)'也写入共享脉络——让代理看到彼此踩过的雷，省下重工成本。

**对曦和的最直接启示**：当前signal_dispatcher是中央信号分发(按Agent Card events_listens匹配)，可以增加DeLM风格的共享验证脉络通道作为补充。gain_landscape.json可以升级为曦和的'共享增益场/共享脉络'混合体。

## 🏗️ 代谢融合方案

### 融合A：存档式进化引擎 (Archive-Based Evolution Engine)

**当前状态**: evolution_engine v3 是线性进化管线
**目标**: 升级为存档式进化引擎，将treasure库作为进化存档

```
当前:   父 → 子 → 孙 (线性链)
目标:   treasure库 (archive) ← 随机/加权采样 → 变异 → 验证 → 回库
        每个treasure都可以是'进化母本'
```

**具体方案**:
1. evolution_engine新增 `sample_parent_from_archive()` 方法，参考DGM的HighScore×LowOffspring

4. signal_dispatcher增加共享脉络通道作为补充分发路径
