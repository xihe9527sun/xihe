## 核心分类法：记忆三分类

| 类型 | 定义 | 曦和映射 |
|:----|:-----|:--------|
| **情景记忆 (Episodic)** | 时间有序的事件日志 | daily logs / chronicles |
| **语义记忆 (Semantic)** | 当前相信为真的知识 | MEMORY.md / cortex / 超图核 |
| **程序记忆 (Procedural)** | 如何做事的指令 | SKILL.md / SOUL.md / IDENTITY.md |

## 关键设计模式

1. **上下文窗口三级管理**：工作记忆→会话压缩→长期持久
2. **混合架构标准**：向量+知识图谱（先语义检索→再图遍历→最后重排序）
3. **声明式记忆注入**：CLAUDE.md/AGENTS.md等文本文件
4. **LLM自主管理记忆**：Agent通过函数调用自主决定存储/检索/遗忘

## 记忆持久化方案对比

| 方案 | 代表 | 优点 | 缺点 |
|:----|:----|:----|:----|
| 纯向量 | Pinecone/Weaviate | 模糊语义检索 | 无法关系推理 |
| 纯图 | Neo4j | 确定性关系推理 | 需维护本体 |
| 混合(向量+图) | Mem0/Zep/Graphiti | 兼顾 | 架构复杂 |
| 文件系统 | CLAUDE.md | 简单轻量 | 规模有限 |

## 对曦和的启示

- 曦和记忆系统天然符合三分类 → 证明方向正确
- 缺少双时态标注 → 待改进（日常日志）
- 程序记忆缺少Agent可写性 → 曦和应该能自主创建SKILL
- 记忆污染防御 → 免疫系统扩展边界
- 当前纯图架构在<20MB规模下足够 → 暂不需向量化
