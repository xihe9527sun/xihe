## 二、觅食战果（3 篇真缺口精品）

### 1. PMD — Procedural Memory Distillation
- **出处**：arXiv:2607.01480（Salesforce Research）
- **机制**：在线构建三层程序性记忆（经验L0 / 洞察L1 / 行为L2），通过「策略-记忆协同进化」蒸馏进策略权重；记忆条件自教师(memory-conditioned self-teacher)读取组合记忆作上下文指导策略
- **实证**：Qwen3-8B 比 SDPO 提升 3.8–5.5%(SciKnowEval) / 7.9–13.6%(LiveCodeBench)
- **真缺口定位**：填补「记忆系统·程序性记忆蒸馏」——现有 claude-mem/automem 管「记忆写入时机」，PMD 补「跨情节程序性规律 → 权重内化」机制（推理时无外部记忆，知识内化进权重）
- **入库状态**：2026-07-18 已入库（本轮重命中判 duplicate，不重复）

### 2. ACP — Agent Context Protocols Enhance Collective Inference
- **出处**：arXiv:2505.14569
- **机制**：领域/智能体无关的结构化协议家族。核心 = 持久化 Execution Blueprint（DAG 编码子任务依赖 + 存中间 agent 输出，下游按依赖复用）+ 标准化消息 schema（AGENT_REQUEST / AGENT_RESPONSE / ASSISTANCE_REQUEST）+ 描述性错误码容错（局部化失败不崩全局）
- **实证**：AssistantBench 长周期网页助手 28.3% SOTA，超商业系统
- **真缺口定位**：填补「多Agent编排·任务上下文传递」——signal_dispatcher 管「按能力分发」，ACP 补「上下文随任务跨 agent 边界流动 + 失败隔离」的执行语义层（盘古点名种子）
- **入库状态**：2026-07-20 本轮新入库 ✅

### 3. ReMe — Remember Me, Refine Me
- **出处**：arXiv:2512.10696（ACL'26 Findings）
- **机制**：动态程序记忆框架，经验驱动智能体进化。三面：①多角度蒸馏（成功模式/失败分析/比较洞察，LLM-as-Judge 校验）②上下文自适应重用（场景感知索引）③基于效用的精炼（自主添加有效记忆 + 剪枝过时，维持紧凑经验池）
- **实证**：Qwen3-8B + ReMe 超越无记忆 Qwen3-14B，证实自进化记忆是计算高效终身学习路径
- **真缺口定位**：补 PMD 未覆盖的「记忆精炼/剪枝」维度（PMD 蒸馏进权重推理无记忆；ReMe 保留可复用记忆池并主动治理）
- **入库状态**：2026-07-20 本轮新入库 ✅
