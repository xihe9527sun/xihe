## 一、ProbMoE 的数学核心

ProbMoE 将 MoE 的离散 top-k 路由重新表述为**基数约束子集分布上的精确概率推断**：

```
传统: logits → softmax → 硬选Top-K
ProbMoE: logits r_j → sigmoid p_j = σ(r_j) → 条件化 |S|=k → 
         精确计算 Z_k = Σ_{|S|=k} Π p_j Π(1-p_j) → 
         解析边际 m_j = ∂log Z_k / ∂log p_j → 可微训练
```

关键使能技术：**SIMPLE估计器** — 通过动态规划在 O(Nk) 时间内精确计算 Z_k，替代了 O(C(N,k)) 的组合爆炸。

## 二、与曦和 TS 路由的对比

| 维度 | 曦和 Beta-Bernoulli TS | ProbMoE (ICML 2026) |
|:-----|:----------------------|:-------------------|
| **范式** | 概率采样 (heuristic) | 概率推断 (theoretical) |
| **路径关系** | 独立后验，无竞争 | 条件子集分布，全局竞争 |
| **梯度** | 无梯度（纯在线学习） | 解析边际梯度（可微训练） |
| **更新方式** | α+=1 (成功) / β+=1 (失败) | 反向传播通过边际概率更新 |
| **探索机制** | Beta后验的不确定性驱动 | 条件概率分布的内在随机性 |
| **复杂度** | O(N) 采样 | O(Nk) SIMPLE归一化 |
| **适用场景** | 在线学习/非可微 | 批训练/可微 |

## 三、洞见链（曦和适用）

### 洞见①：路径之间的竞争关系必须显式建模

当前问题：Beta-Bernoulli TS 每条路径独立更新。路径 A 获得成功→α+=1，路径 B 什么都没发生。但实际上 B 是因为 A 被选中而**被放弃**的，B 也应该有更新。

**解决方案：负选路径的隐式更新**
- 被选中的路径→正常更新 (success=True)
- 未被选中的路径→隐式失败 (success=False, boost=False)
- 这样"被选即赢，未选即输"形成了路径间的间接竞争

### 洞见②：探索路径需要差异化先验

当前问题：新路径用 Beta(1,1) 无信息先验——首次成功即 α=2, β=1 → 期望值=0.67。但首次成功可能是偶然的。

**解决方案：保育期差异化先验**
- 新注册路径用 Beta(0.5, 0.5) —— Jeffreys先验，比无信息先验更"审慎"
- 来自变异池(v2)的路径用 Beta(1, 2) —— 悲观先验，需要更多证据确认
- 来自代谢路由探索的路径用 Beta(2, 1) —— 乐观先验，鼓励持续尝试

### 洞见③：惊奇度驱动的β衰减替代定期全量衰减

当前问题：TS_BETA_DECAY_BATCH=100 时全量衰减所有路径的β。这导致冷路径和热路径被无差别对待。

**解决方案：惊奇度驱动的差异化衰减**
- 当一条路径带来高惊奇度（预测误差>阈值）且未被选中时→给予β额外衰减
- 高惊奇意味着"系统预期这条路径不行，但它可能行了"→给予重新探索的机会
- 低惊奇冷路径→维持原状
- 这比 ProbMoE 更轻量，但抓住了"动态重置"的核心精神
