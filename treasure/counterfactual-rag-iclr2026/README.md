## 一、核心发现：相关性陷阱(Correlation Trap)

论文识别出RAG系统的根本漏洞——**相关性陷阱(Correlation Trap)**：

> 现有RAG系统**无法区分"因果决定性证据"与"高度相关但误导性信息"**。

即使语义相似度很高，检索到的片段可能只是与答案相关、但不是因果决定因素。这种系统性漏洞在所有RAG系统中都存在。

## 二、核心方法：CF-RAG的双机制

### 机制1：反事实查询生成
系统为每个查询自动生成反事实变体：
- "如果不考虑X会怎样？"
- "如果把X替换为Y呢？"

然后评估检索结果对反事实查询的鲁棒性——如果一个片段在反事实变体下仍然被检索到，那它包含的是相关性噪音；只有在反事实变体下不再被检索到的，才是真正因果性证据。

### 机制2：平行仲裁
通过并行处理机制调和冲突证据，不增加检索开销，只在推理阶段增加反事实评估层。

### 性能
- 在挑战性基准上显著提升鲁棒性
- 保持与标准RAG相当的计算效率

### 与CausalFlow的协同

```
CF-RAG: 输入验证(检索时去伪存真)

```

三者构成完整的"输入验证→执行诊断→输出验证"因果闭环。
