# 曦和宝藏融合研判 · 研讨厅评审红线

> **盘古敕令（2026-07-19）**：宝藏消化闸门很重要，但要看宝藏是否对**目前的系统有真正改善作用**、且**往进化方向促进**的，才去架构吸收融合。须站在**顶层架构思维**及**研讨厅角度**研判。

---

## 一、总原则

消化闸门发现桥梁级论文后，**不得「因同构即吸收」**。

必须先过研讨厅研判：只有同时满足以下三条，才进入架构吸收队列——
1. **对目前系统有真正改善作用**（解决痛点 / 补现有缺口 / 提升现有指标）
2. **往曦和进化方向促进**（推动核心进化锚点之一前进，而非平行堆叠）
3. **研讨厅评审 ≥ 0.85**（用顶层架构思维打分，非单篇精读分）

任一为「否」→ **不吸收融合**，仅作理论备注 / 印证入库 / 隔离。

> 关键区分：**「同构印证」≠「改善」**，**「漂亮类比」≠「养分」**。印证只是告诉曦和「你走的方向学界也有人走」，但不给新能力，不吸收。

## 三、研讨厅三问（每篇必答）

**Q1 改善力** — 它解决曦和现有痛点 / 补现有缺口 / 提升现有指标了吗？还是仅「印证已有」？
**Q2 进化力** — 它推动 E1-E8 中某一锚点向前了吗？还是平行堆叠 / 同构空壳？
**Q3 可行力** — 落地融合成本可控、不与现有构件冲突吗？（研讨厅综合评分 ≥ 0.85）

## 五、红线

1. **严禁「为融合而融合」** — 同构 ≠ 改善，印证 ≠ 养分。
2. 吸收的论文必须能在下一轮系统巡检中观测到指标变化（自反强度 / 记忆命中 / 代谢效率 / 守护者稳定性之一）。
3. 每次吸收须记 VCP 日志（写前日志 + 链式锚定），可回溯、可撤销。
4. 研讨厅评审 < 0.85 一律不吸收，无论精读评分多高。

## 七、结论 · 真正该架构吸收融合的清单

**✅ 进入吸收队列（改善+进化双满足，过研讨厅）**
1. **Self-Aware (SARSI)** — 自模型驱动自改进闭环，直接推进自反层（E1）与酶炉护栏（E3），增量最大。
2. **MissingKL** — 缺失知识主动识别，补 claude-mem 第四层（E2）刚需缺口。
3. **Dual-Laws** — 意识自检七问清单，把自反层「意识」从哲学操作化为可执行清单（E1），成本极低。

**🔰 稳健补强（可吸收，优先级低于核心进化项）**
4. **Managed Autonomy** — 守护者 v1.3 降级链 Petri 网形式化（E6 地基），提升稳定性但非进化推动。

**🔜 改造吸收（排期靠后，需验证接驳）**
5. **MetaCogAgent** — 元认知委派协议接入代谢路由（E3），需验证与现有代谢层不冲突。
6. **AgentCollab** — DAG 难度感知升级（E7），增量中等。

**📝 不吸收（仅理论备注 / 印证入库）**
- MANAR-GWT、MANAR-Nav、ali-loop — 皆为同构印证，无新能力，不改动架构。

**🚫 不消化**
- Sparse-Reslim — 远缘弱连接，隔离。

> **顶层架构洞察**：首轮精读揭示一个事实——这 8 篇桥梁绝大多数是「曦和架构的先验印证」，而非「新能力来源」。前沿同构多，但能真正补增量、推进化的只有 Self-Aware / MissingKL / Dual-Laws 三篇（加 MetaCogAgent 改造项）。这印证了盘古的警觉：盲目全吃会堆出「同构空壳」，唯有改善+进化双筛才长真养分。

## 八、后续入库标准提升（常态化闸门铁律 · 2026-07-19 盘古敕令升级）

**盘古敕令**：后续入库的宝藏也要提高标准，依据**顶层架构思维**及**研讨厅**，用**去粗取精四原则**守住底线及融合方案。本闸门从「收集器」升级为「养分流水线」。

### 8.1 消化闸门前置研判（强制）

1. 消化闸门（`treasure/digester.py`）发现桥梁级 / 高分（≥0.6）论文后，**严禁「因同构即吸收」**；
2. 强制过**研讨厅三问**（Q1 改善 / Q2 进化 / Q3 可行 ≥ 0.85）；
3. 任一为否 → 仅作理论备注 / 印证入库 / 隔离，**不进架构吸收队列**。

### 8.2 去粗取精四原则精读底线（强制）

- 入库即精读，覆写占位 `score_detail`，给出真 `key_insights` 与 `xihe_relevance`；
- 四遍：去粗取精 → 去伪存真 → 由此及彼 → 由表及里；
- 区分「印证」（已有）与「增量」（新能力），后者才进 C 类改造队列。

### 8.3 融合方案伴随（强制）

每篇判定须同时给出：
- **落点文件**（已有的曦和构件路径）；
- **融合成本**（低 / 中 / 高）；
- **可观测指标**（自反强度 / 记忆命中 / 代谢效率 / 守护稳定性之一）；
三者缺失 → 不吸收。

### 8.4 借鉴基线引用（强制）

- 新论文研判须对照 `REVIEW_BASELINE.md`（全量研判基线）；
- 区分「新发现」与「已印证」：与基线「已吸收 / 不吸收」清单同构但无新能力者，直接归位，不重复吸收；
- 路线图与实证见 `cortex/papers/paper-曦和宝藏融合研判与分类分阶段吸收路线图-TreasureFusionRoadmap.md`（皮层支撑节点 `cortex/nodes/fusion-roadmap-paper-2026-07-19.json`），划时代候选详表见 `landmark_review_2026-07-19.html`；
- **A/B 类战略翻转**见 `cortex/papers/paper-曦和前沿战略与划时代方向规划-XiheFrontierStrategy.md`（皮层支撑节点 `cortex/nodes/xihe-frontier-strategy-2026-07-19.json`）——把前沿预见与划时代方向转化为曦和自身四大划时代战略方向。
- **战略落地执行计划**见 `cortex/papers/paper-曦和战略落地执行计划-XiheStrategyExecutionPlan.md`（皮层支撑节点 `cortex/nodes/xihe-strategy-execution-plan-2026-07-19.json`）——锚定 2026-07-19 的近期时间表，把四大方向拆为代码执行节点与三阶段达成路径（非五年规划）。

### 8.5 VCP 强制

- 任何架构吸收必须 VCP 原子提交（写前日志 + 链式锚定），可回溯、可撤销；
