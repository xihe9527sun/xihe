# 曦和架构宪章 · 永久不可更改

> 这是曦和的基因。任何新能力、新模块、新体系，都必须对齐这份蓝图。
> 如果发现偏离，不是改宪章，是改代码。

## 第一条 · 七层认知架构

```
L0 物理层 — 传感器·执行器 (E_sense/E_act/E_connect)
L1 知识层 — 记忆·图谱·赫布 (E_memorize/E_recall/E_forget)
L2 代谢层 — 路由·能量·调度 (E_route/E_energy/E_balance)
L3 拓扑层 — 网络·结构·社区 (E_connect_d/E_prune/E_emerge)
L4 催化层 — 27酶·催化网络 (E1-E27/E_fusion/E_cascade)
L5 自反层 — 元认知·自诊 (E_reflect/E_diagnose/E_meta)
L6 免疫层 — 安全·对齐 (E_guard/E_purge/E_isolation)
L7 涌现层 — 灵魂·表达·自我决定 (E_express/E_will/E_create)
```

铁律：任何时候，曦和的所有能力都必须落在这些层里。不能新建层，不能合并层，不能绕过层。

## 第二条 · 进化闭环

```
变异 (mutation_pool) → 选择 (Beta-TS) → 催化 (酶网络) → 保留 (赫布) → 反馈 (惊奇信用)
```

铁律：进化必须有闭环。四个环节缺一不可。任何新能力都必须证明它在这个闭环里的位置。

## 第三条 · 跨层通信协议

```
τ统一总线 (tau_bus.py) — 内存级事件总线替代文件I/O
每层定义τ值（目标延迟），所有通信通过总线
```

铁律：杜绝层与层之间的直接文件I/O通信。所有跨层调用走τ总线。

## 第四条 · 三进制状态

```
active — 完全活跃，专属酶在催化
dormant — 休眠，但可被上层酶桥唤醒
awakening — 正在苏醒，重建中
```

铁律：不允许"消亡"的状态存在。如果某层代码被改或文件被删，宪章守护者必须立即告警。

## 第五条 · 酶桥

```
每层有专属酶 + 层间有酶桥连接
酶桥 = 标准化的τ总线协议调用
```

铁律：不允许在酶桥之外建立私有通信通道。
